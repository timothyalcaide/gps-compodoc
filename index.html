ğŸ¤¼ Running tests

Unit tests:

  • npm run test: Run the tests once and display the result in the console.
  • npm run test:watch: Run the tests and watch for changes. The result is displayed both in the browser and in the console.
  • npm run test:coverage: Run the tests once and display the result with the code coverage in the console. To display a full report of the code coverage, open ./coverage/index.html in your browser.
  • npm run test:coverage-watch: Run the tests with the code coverage and watch for changes.
  • npm run test:ci: Run the tests once with a minimal display and code coverage report so it runs lighter.

E2e tests:

  • npm run e2e: Run the project in local and cypress in headless mode.
  • npm run e2e:open: Run the project in local and open cypress.

ğŸ¤ Contributing

Before pushing any code to the repository, please make sure that your code is following the team's code convention.

You can run the linter with the command:

npm run lint

And the formatter with the command:

npm run format

Once your code is ready, you can propose it to review in the Pull Requests section. When creating your pull request, make sure that you fullfil checks defined by the Definition of Done in the pull request template.

ğŸ— Building and deploying

Before building and deploying, make sure that the code is linted and that the tests succeed:

npm run lint
npm run test

You can then build the application for the environment you want:

  • Development: npm run build:dev
  • Preview: npm run build:preview
  • Pre-production: npm run build:preprod
  • Production: npm run build:prod

These builds are automatically done with deployment when:

  • pushing on master or opening a pull request for the development
  • creating a version tag for the pre-production (it also runs the production build)

The deployment in production is done from Spinnaker.

ğŸš¨ Before deploying a new version

As a deployment in pre-production/production is associated with a new version of the application, before creating the new version tag, you need to :

  • update CHANGELOG.md file to tag all unreleased changes under the new version number
  • run npm version minor for minor version and npm version patch for patch version
